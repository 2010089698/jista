# MVP要件定義\_Jista

## 目的

参加するオリエンテーリングイベントのスタート時刻を簡単に把握できる

## 成果指標

4週間継続率40%以上

## コアユーザー

スタートリストや自分のスタート時刻を探すのが面倒くさいオリエンティア

## 主要ユースケース

- 事前に氏名を登録しておく  
- スタート時刻を知りたくなったらアプリを開く  
- 参加するイベントを選ぶとスタート時刻が表示される  
- 一度選んだ未来のイベントのうち最も日付が近いもののスタート時刻はオフラインでもすぐに表示される

## スコープ

- やる：イベント選択、スタート時刻抽出・保存、オフライン表示  
- やらない：通知、移動時間逆算、カレンダー連携、OCR

## 画面遷移

起動→メイン画面

## 状態

- オフライン: ローカルデータ即表示  
- オンライン: スタート時刻取得可能

## データ

- スタート時刻取得時に保存するもの  
  - イベント名  
  - 開催日  
  - スタート時刻  
- スタート時刻取得時に継続率計測のために匿名IDをサーバー送信  
- 直近5件を端末に保存（古いものから消す）

## オフライン・同期方針

- オフライン: 選択済みのイベントのスタート時刻が表示可能  
- オンライン: Japan-O-Entryの大会ページを元にイベントを選択、スタート時刻取得

## 権限・通知設計

- 権限は求めない  
- 通知なし

## 非機能要件

- 起動性能: コールドスタート ≤ 3秒（P50）、画面遷移 ≤ 1秒（P50）  
- 対応環境: iOS 16+ / Android 10+

## 計測・判定ルール

継続率

- 定義: 初回スタート時刻取得から28日後に、同じ匿名IDで1回以上スタート時刻取得を実施  
- 計算: 継続ユーザー数 ÷ 初回利用ユーザー数

## リリース・ロールバック戦略

- 段階的リリース: まず数%のユーザーだけに公開し、問題なければ範囲を広げる  
- ロールバック: 致命的な不具合があれば、機能単位で無効化

## セキュリティ・プライバシールール

匿名IDのみ取り扱う

## 技術的前提条件

- expo SDK 54  
- EASなしでexpo goで起動できる  
- サーバがJapan-O-Entry を見に行ってイベントリストをJSONでアプリに返す  
- サーバがJapan-O-Entry を見に行ってスタートリストPDFを取り、Pythonで時刻を抜きアプリに返す
